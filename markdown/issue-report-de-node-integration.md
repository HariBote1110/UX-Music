# ISSUE: De-node-integration 移行後の表示・再生不具合

## 概要
セキュリティ強化のための De-node-integration（`nodeIntegration: false`, `contextIsolation: true` への移行）実施後、レンダラープロセスにおいて以下の重大な不具合が発生しています。

1. **楽曲リスト（トラックビュー）が空になる**: 「For You」セクション以外で曲が1曲も表示されない。
2. **楽曲の再生ができない**: 曲を選択しても再生が開始されない。
3. **IPC 通信のエラー**: 開発者ツールにおいて `undefined` 参照のエラーが散見される。

## 調査済みの原因と現状
### 1. IPC ホワイトリストの不備
`src/main/preload.js` で定義した `contextBridge` のホワイトリストに、以前は暗黙的に許可されていた多数のチャネルが漏れていました。
- **対応**: 主要なチャネル（約60個）をホワイトリストに追加しましたが、まだ不足している、あるいは名称が間違っているチャネルが存在する可能性があります。

### 2. コールバック引数 `event` の不整合
Electron の `ipcRenderer.on` は `(event, ...args)` を渡しますが、`preload.js` でのラッピング時に `event` を除去するか渡すかでレンダラー側の挙動が壊れています。
- **現状**: `(event, ...args) => func(...args)` とし、レンダラー側は `(data) => ...` で受け取るように統一を試みましたが、一部のモジュールで引数の数が合わずに `data` が `undefined` になる現象が続いています。

### 3. イベント名称の不一致
メインプロセスから送信されるイベント名（例: `load-library`）と、レンダラーが期待しているイベント名（例: `library-loaded`）が混在しており、初期化フローが正常に完了していない可能性があります。

## 実施した主な変更
- `src/main/preload.js` の新設。
- レンダラー全域からの `require('electron')`, `require('path')` 等の排除。
- `src/renderer/renderer.js` 等の IPC リスナー host 書き換え。

## 今後の修正方針（リコメンデーション）
1. **IPC チャネルの完全同期**: `src/main/ipc-handlers.js` および各 `handlers/*.js` で定義されている `send` / `handle` 名と、レンダラーの `on` / `invoke` 名を一つずつ突合し、完全に一致させる必要があります。
2. **引数形式の完全統一**: 全ての `electronAPI.on` コールバックを `(event, data) => ...` にするか `(data) => ...` にするかを決定し、全ファイルを機械的に修正する必要があります。現在は混在が原因でエラーが起きています。
3. **初期化シーケンスのデバッグ**: `renderer.js` の `initApp` 内で、設定 of ロード、ライブラリのロード要求、およびそれらのレスポンス受信の順序が正しく守られているか、ログを詳細に出して確認する必要があります。

## 備考
作業は `preload.js` のホワイトリスト更新と主要レンダラーファイルの引数修正まで完了していますが、想定以上に依存関係が複雑であり、一斉置換によるデグレードが発生している状態です。
